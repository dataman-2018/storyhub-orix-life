採用広報パッケージ提案

社内の語りを、
採用の武器にする。

年間100本超の社内報コンテンツ。社員の声、部門の挑戦、経営の想い——
貴社が積み上げてきた語りの資産を、採用広報として社外に届ける。
StoryHubが、その転用パイプラインを設計します。

01 — About This Proposal

この提案について

貴社は既にStoryHubを活用し、社内報の制作体制を構築されています。
本提案は「新しいツールの導入」ではなく、既存の運用資産を採用広報に拡張するための提案です。

2026年4月21日の定例会議でお聞きした内容を踏まえ、貴社が直面する3つの具体的な課題に対して、StoryHubプラットフォームの既存機能と新機能を組み合わせた解決策をご提案します。

1

語りの設計

社内報のカテゴリー整備、ヒアリング項目の標準化、編集方針の策定。記事の目的と設計を先に定める

2

取材の自動化

自動インタビュー機能により、社員取材の品質を均一化し、外部ライター依存を解消する

3

社外への展開

社内報で蓄積した語りの資産を、採用サイト・採用コンテンツとして社外に転用するパイプラインを構築

02 — Issues

なぜ今、採用広報を
内製化すべきなのか

貴社の広報チームが抱える構造的な課題は、個別の努力では解決しにくい、プロセスそのものに内在する問題です。

課題1 — 外部ライター委託の品質問題

採用サイトの社員インタビューは外部ライターに委託していますが、「書いてほしい内容を伝えきれず、半分以上を書き直す」状況が続いています。書き直しに費やす工数は、実質的に「二重の制作コスト」を意味します。

この問題の根本は、外部ライターに「何を聞くべきか」「どう書くべきか」の基準が体系化されていないことにあります。結果として、担当者の属人的な判断に品質が依存し、制作の再現性が確保できていません。

課題2 — 社内報資産が社内で眠っている

年間100本超の社内報を制作する過程で、社員の想い・部門の取り組み・経営施策の背景といった豊富な一次情報が日々生まれています。しかし、これらの素材は社内メール配信の中だけで消費され、社外に転用されていません。

秋谷様が言及された「採用広報が先行してできそうな領域」という認識は正確です。社内報の中には、採用候補者が「この会社で働く意味」を知るために最も価値のある情報が既に存在しています。

課題3 — カテゴリー・ヒアリング項目・レシピの未整備

志村様よりご説明いただいた今年度の重点課題——「カテゴリー・ヒアリング項目・プロンプト（レシピ）の三点整備」——は、社内報の品質向上と採用広報転用の両方に直結します。

記事の目的分類（エンゲージメント向上、挑戦促進等）と形式分類（対話形式、一人称等）のどちらを軸にするかは、コンテンツの受け手（社員なのか候補者なのか）によって変わります。この設計を先に固めることで、レシピとヒアリング項目が自然に導かれます。

構造的に同型の問題：「メディア立ち上げに近いプロセス」という定例会議での議論が示す通り、貴社の社内報は「編集方針なきまま制作が進む創刊期のメディア」と同じ構造を持っています。編集方針を先に定めることで、コンテンツの種類・KPI・レシピ設計がトリクル構造で整理できます。
03 — Concept

社内報を、
採用広報のエンジンにする

語りの手綱を、自らが引いていく。
外注依存から自走体制へ。社内に蓄積した語りの資産を、採用候補者に届くかたちに変換します。

強度の高いナラティブは、一時的な取り組みではなく、
継続的な仕組みから生まれる。

本提案の設計思想は、「語りの設計 → 制作 → 公開 → 蓄積 → 学習」の循環サイクルを社内に実装することです。コンテンツ制作代行（BPO）ではなく、貴社が「語りの主体」となるための体制構築支援（BPOwning: 自走）を最終ゴールとします。

課題 本プログラムによる解決
外部ライター依存 自動インタビュー＋社内レシピ整備で、取材から記事化まで内製化。書き直し工数をゼロに近づける
社内報資産の死蔵 社内→社外の転用パイプラインを構築。同一の取材素材から社内報と採用記事を同時生成
カテゴリー未整備 編集方針策定支援により、目的別カテゴリー・ヒアリング項目・レシピを一括設計
既存StoryHub利用との連続性：本プログラムは、貴社が既にStoryHubで運用している社内報制作フローの延長線上にあります。新たなツール導入や学習コストは最小限に抑え、既存のレシピ・素材・ワークフローを採用広報向けに拡張する設計です。
04 — Program

3つのフェーズで、
段階的に立ち上げる

いきなり最初から自走はできません。編集方針の策定から始め、段階的に社内に「語る力」を移していきます。

Phase テーマ 期間 主な内容
Phase 0 編集方針策定 1〜2ヶ月目 カテゴリー設計・ヒアリング項目標準化・レシピ一括整備
Phase 1 自動インタビュー導入 3〜4ヶ月目 AIインタビュー機能の実装・社員取材の自動化
Phase 2 社外展開 5ヶ月目〜 社内報→採用コンテンツの転用パイプライン構築・運用開始
Phase 0 — 1〜2ヶ月目

編集方針策定：語りの基盤をつくる

StoryHubが懇意とする広報・編集プロフェッショナルと連携し、社内報と採用広報を貫く編集方針を策定します。志村様が指摘された「カテゴリー・ヒアリング項目・レシピの三点整備」を、このフェーズで一括実現します。

成果物 内容
コンテンツカテゴリーマップ 社内報と採用広報を横断する目的別カテゴリー体系。「経営起点の深掘り」「部門起点の紹介」「採用候補者向け転用」の三層で整理
標準ヒアリングシート カテゴリー別の取材質問テンプレート。「具体例の引き出し」「感情の変遷」の二点を必ず含む設計
レシピセット カテゴリー対応のStoryHubレシピ。社内報版と採用広報版を同時に整備
編集ガイドライン トーン＆マナー、文字数基準、配信チャネル別のフォーマット規定
Phase 1 — 3〜4ヶ月目

自動インタビュー導入：取材を自動化する

StoryHubの自動インタビュー機能を導入し、社員取材の品質を標準化します。外部ライターへの委託を段階的に解消し、取材から記事化までの内製体制を構築します。

活用シーン 内容
部門起点の取り組み紹介 各部門から依頼のある「新しい取り組み紹介」を、AIインタビューで効率的に取材。年間発信本数の維持・拡大に貢献
採用向け社員インタビュー Phase 0で設計した標準ヒアリングシートに基づき、具体例と感情の変遷を必ず引き出す構造化インタビューを自動実施
経営層メッセージ 月1本の経営起点コンテンツの取材補助。対面取材前の論点整理や、フォローアップ取材に活用
品質担保の設計：秋谷様・志村様のご指摘の通り、カテゴリーとヒアリング項目が先に固まっていれば、AIインタビューの品質は担保できます。Phase 0の編集方針策定が、Phase 1の品質基盤になります。
Phase 2 — 5ヶ月目〜継続

社外展開：社内報を採用コンテンツに転用する

社内報の制作過程で蓄積された素材・記事を、採用候補者向けコンテンツとして再構成して公開します。StoryHubの「1ソースマルチユース」機能により、同一の取材素材から社内報と採用記事を同時生成する運用を確立します。

社内報カテゴリー 採用広報への転用
部門の取り組み紹介 「チームで働く」シリーズ — 部門の雰囲気・仕事の進め方を候補者向けに再構成
経営施策の深掘り 「経営者の思い」シリーズ — 候補者が「経営の方向性」を理解するための記事に変換
社員の挑戦・成長記録 「入社後のリアル」シリーズ — キャリアパス・成長環境を伝える採用コンテンツに
新制度・働き方紹介 「オリックス生命の働き方」 — 福利厚生・制度を候補者の関心に即して紹介
既存の配信基盤との接続：現在の社内報は全社員に毎朝9時メール配信されています。採用広報コンテンツは採用サイトへの掲載を起点に、適宜SNS・採用媒体へ展開します。社内報と採用広報で配信チャネルは分離しつつ、素材は共通化します。
05 — Platform

StoryHubプラットフォームの
拡張活用

貴社が既に利用しているStoryHubの機能に加え、新機能を組み合わせて採用広報の体制を構築します。

Studio

既にご利用中
社内報レシピ・ニュースレター生成で活用中。採用広報向けレシピの追加、カテゴリー別テンプレートの整備で、社内→社外の転用ワークフローを実装

Interview

新規導入（6月〜）
AIによる構造化インタビュー。Phase 0で設計した標準ヒアリングシートに基づき、社員が自律的に回答。具体例・感情の変遷を確実に引き出す

Brand Memory

拡張活用
編集ガイドライン・カテゴリーマップ・過去素材を一元管理。担当者が異動しても語りの基準が断絶しない。社内報と採用広報の一貫性を担保

Version & Folder

リニューアルで追加
コンテンツ・素材のフォルダ管理、編集履歴の全バージョン記録。以前からのご要望を実現。社内報/採用広報のコンテンツを整理して管理

1ソースマルチユース：同じ取材素材から複数コンテンツを生成

StoryHubリニューアルにより、過去にアップロードした素材や既存コンテンツをチェックボックスで選択し、新記事の素材として再利用できるようになりました。これにより、1回の社員インタビューから社内報記事と採用コンテンツを同時に生成する運用が可能です。

1
取材素材のアップロード
AIインタビュー or 対面取材の音声・テキストをStoryHubに格納
2
社内報記事の生成
社内報カテゴリー対応レシピで記事を生成。ニュースレター版（短尺ベタ貼り）に最適化
3
採用コンテンツの生成
同じ素材を選択し、採用広報向けレシピで変換。候補者の関心に合わせてトーンと構成を調整
4
レビュー・公開
広報チームが品質確認の上、社内報はメール配信、採用コンテンツは採用サイトへ公開
06 — Training

自走化支援：
語りの力を社内に移転する

外注依存から自走体制へ。制作を代行するのではなく、貴社の広報チームが自ら語り続ける力を育てます。

月1回の勉強会・ワークショップを通じて、編集方針の運用、AIツール活用、コンテンツ品質の向上を段階的に進めます。座学ではなく、実際のコンテンツを題材に「書きながら学ぶ」実践形式です。

回 テーマ 内容
第1〜2回 編集方針の実装 Phase 0で策定したカテゴリー・ヒアリング項目を実運用に落とし込む。レシピの使い分け演習
第3〜4回 インタビュー設計 「具体例の引き出し」「感情の変遷」を軸にした取材技法。AIインタビューの活用実践
第5〜6回 社内→社外の転用技術 同一素材から社内報と採用記事を作り分ける編集技術。読み手の違いに応じたトーン調整
第7〜8回 AIツール活用の深化 StoryHub新機能の実践活用。素材再利用、バージョン管理、フォルダ管理の最適運用
第9〜10回 品質管理・KPI設計 コンテンツ品質チェックの仕組み化。採用広報の効果測定指標の設計
第11〜12回 自走計画の策定 社内ナラティブガバナンスの設計。担当者引き継ぎ・体制の恒久化。今後のロードマップ
