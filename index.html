01 — About This Proposal

この提案について

本提案は「新しいツールの導入」ではありません。貴社が既にStoryHubで構築した社内報制作フローを基盤に、CCチーム全体の編集体制を段階的に強化するための支援プログラムです。

4月21日の定例会議でいただいたキックオフ資料「経営広報の実践とこの指止まれを生み出す一年へ」、および志村様からご説明いただいた今年度方針を踏まえ、CCチームの4領域（社内広報・社外広報・ウェブサイト・CI）にStoryHubがどう貢献できるかを整理しました。

1

編集方針の策定

カテゴリー・ヒアリング項目・レシピの三点整備。貴社の既存整理をベースに、プロの編集知見で精緻化する

2

AI編集体制の構築

自動インタビューとレシピ設計で「部門Push → CCレビュー」の半自動化プロセスを実現する

3

社外への展開

社内報で蓄積した語りの資産を、採用広報・ウェブサイトへ転用するパイプラインを構築する

02 — Issues

課題は「やっていない」ではなく、
「つながっていない」こと

貴社キックオフ資料の言葉を借りれば、課題は発信量ではなく「接続」にあります。点在する発信を線にし、経営の方向性と現場の挑戦をつなぐ編集体制が必要です。

課題1 — カテゴリー・ヒアリング項目・レシピの未整備

志村様が今年度の重点課題として挙げられた「カテゴリー・ヒアリング項目・プロンプト（レシピ）の三点整備」は、社内報の品質向上のみならず、社外展開・AI活用の前提条件です。記事の目的分類が定まらなければ、AIに何を書かせるかも定まりません。

貴社は既にカテゴリーの言語化に着手されています。本プログラムでは、その既存整理を出発点に、編集プロフェッショナルの知見を加えて体系化します。ゼロから作るのではなく、貴社の手元情報をベースに精緻化するアプローチです。

課題2 — 「部門Push」を支える仕組みがない

キックオフ資料のPhase 1は「部門が記事初稿を提出 → CCがレビューして発信」の体制をAI支援で構築することです。しかし現状は、ヒアリング項目が属人的で、取材品質にバラつきが出やすい。「具体例の引き出し」「感情の変遷」が情報として揃わなければ、記事化の精度は上がりません。

自動インタビュー機能は、この「部門Push」を構造化する仕組みです。標準化されたヒアリング項目に基づき、各部門の社員が自律的に回答し、CCは品質レビューに集中できます。

課題3 — 社内の語りが社外に届いていない

年間100本超の社内報は、豊富な一次情報の宝庫です。しかし採用サイトの社員インタビューは外部ライターに委託し、「書いてほしい内容を伝えきれず、半分以上を書き直す」状況が続いています。

キックオフ資料で示されたウェブサイト戦略——「コンテンツを最重要視し、経営・人・挑戦の物語を蓄積する」——を実現するには、社内報で生まれる一次情報を社外コンテンツに変換するパイプラインが必要です。

03 — Concept

「この指止まれ」を、
編集の仕組みにする

語りの手綱を、自らが引いていく。外注依存から自走体制へ。
StoryHubは制作代行者ではなく、編集体制の共同設計者として伴走します。

経営の方向性のもとで、現場の挑戦が広がる状態をつくる。
その状態を「仕組み」として持続させるのが、編集体制である。

貴社キックオフ資料のミッション——「経営広報（トップダウン）× この指止まれ（ボトムアップ）を接続する設計者」——は、まさに編集の仕事そのものです。本プログラムは、その接続を属人的な判断ではなく、再現可能な仕組みとして社内に実装することを目指します。

課題 本プログラムによる解決
カテゴリー未整備 編集方針策定支援で、目的別カテゴリー・ヒアリング項目・レシピを一括設計。貴社既存の整理をベースに精緻化
部門Pushの仕組み不足 自動インタビュー＋標準レシピで「部門が記事初稿を提出 → CCがレビュー」体制をAI支援で構築
社外展開の未着手 社内→社外の転用パイプライン構築。同一の取材素材から社内報・採用記事・ウェブコンテンツを同時生成
外部ライター依存 自動インタビューと社内レシピ整備で取材から記事化まで内製化。書き直し工数を解消
StoryHubの立ち位置：プロダクトの提供者ではなく、編集体制構築の戦略的パートナー。ツールの使い方を教えるのではなく、「何を・誰に・どう語るか」の設計から共に考え、その実行をAI基盤で支える関係です。

PoC：編集方針策定（3ヶ月）

StoryHubが懇意とする広報・編集プロフェッショナルと連携し、CCチームの編集基盤を策定します。貴社が既に言語化されている編集方針・コンテンツカテゴリの整理を出発点に、プロの知見を加えて体系化。「経営の意向により変化が速い部分」にも対応できる、柔軟かつ構造的な設計を目指します。

成果物 内容
コンテンツカテゴリーマップ 「経営起点の深掘り」「部門起点の紹介」「採用候補者向け」の三層で整理。目的分類と形式分類の両軸を統合
標準ヒアリングシート カテゴリー別の取材質問テンプレート。「具体例の引き出し」「感情の変遷」を必ず含む設計

PoCの位置づけ：PoCは「試用」ではなく、Phase 1以降の品質基盤を作る工程です。ここで策定したカテゴリーとヒアリング項目が、自動インタビューの品質を決定し、社外展開の可否を左右します。PoC完了後、成果をもとに継続支援の要否を判断いただきます。
Phase 1 — AI編集体制の構築

「部門Push × CCレビュー」を仕組みにする

PoCで策定した編集方針をもとに、自動インタビュー機能を導入。各部門の「新しい取り組み紹介」を、AIインタビューで効率的に取材し、CCチームはレビュー・品質管理に集中できる体制を構築します。

「この指止まれ」の仕組み化 StoryHubの支援
部門の挑戦を言語化する 自動インタビューが標準ヒアリング項目に基づき社員から具体例・感情の変遷を引き出す
CCがレビューし発信する カテゴリー対応レシピでAIが初稿を生成。CCは編集・品質確認に集中
他部署の挑戦が見えるようにする 社内報として全社配信。「点をつなぎ、共感・協業を生む」サイクルを支える
経営層メッセージを届ける 月1本の経営起点コンテンツの取材補助。対面取材前の論点整理にも活用
KPIとの連動：今年度の新KPI「能動的ヒアリング本数」は、自動インタビュー機能の導入件数と直接連動します。重点深掘りニュース20-30本に対して、どれだけ能動的に取材できたかを定量的に追跡できます。
Phase 2 — 社外展開・自走化

CCを「発信者」から「増幅者」へ進化させる

社内報の制作過程で蓄積された素材を、採用広報コンテンツ・ウェブサイトコンテンツとして社外に展開。キックオフ資料のPhase 2「広報は発信者から認知を上げる増幅者へ進化する」を実現します。

社内報カテゴリー 社外展開先
部門の取り組み紹介 採用サイト「チームで働く」シリーズ / ウェブサイトの事例コンテンツ
経営施策の深掘り ウェブサイト「経営の方向性」コンテンツ / 採用広報「経営者メッセージ」
社員の挑戦・成長記録 採用サイト「入社後のリアル」シリーズ / オウンドメディア
座談会コンテンツ 採用サイト「社員の対話」/ ウェブサイト（経営・人・挑戦の物語として蓄積）
ウェブサイト戦略との接続：キックオフ資料の「コンテンツを最重要視し、経営・人・挑戦の物語を蓄積して"見つかる資産"にする」方針と直結。社内報→採用広報→ウェブサイトの3段階転用パイプラインにより、1回の取材が複数チャネルの資産になります。
05 — Platform

StoryHubプラットフォームの
拡張活用

貴社が既に利用しているStoryHubの機能に加え、新機能を組み合わせてAI編集体制を構築します。

Studio

既にご利用中
社内報レシピの運用に加え、PoCで設計するカテゴリー別レシピセットを実装。社内報と採用広報の両方に対応するマルチユースワークフローを構築

Interview

Phase 1で導入
AIによる構造化インタビュー。「部門Push」の標準化ツールとして、各部門の社員が自律的に回答。具体例・感情の変遷を確実に引き出す

Brand Memory

拡張活用
編集ガイドライン・カテゴリーマップ・経営の方向性資料を一元管理。担当者が異動しても語りの基準が断絶しない。経営の意向変化にも柔軟に対応

1ソースマルチユース

Phase 2で活用
過去素材の再利用機能により、1回の取材から社内報・採用記事・ウェブコンテンツを同時生成。外部ライター委託を構造的に不要にする

AI編集体制のワークフロー

1
部門からの取材（自動インタビュー or 対面）
標準ヒアリングシートに基づくAIインタビュー、または対面取材の音声をStoryHubに格納
2
AI初稿生成（カテゴリー対応レシピ）
社内報用レシピで自動生成。ニュースレター版（短尺ベタ貼り）に最適化した出力
3
CCレビュー・品質管理
編集ガイドラインに基づきCCチームが品質確認。バージョン管理で編集履歴を完全記録
4
社内配信 + 社外転用（Phase 2以降）
同じ素材から採用広報向けレシピで変換。社内報はメール配信、採用コンテンツはウェブサイトへ
